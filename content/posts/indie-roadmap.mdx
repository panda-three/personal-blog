独立开发不是孤军作战，而是用最短路径验证想法。

## 约束

- 4 周一小版本，先丢出可用版本
- 预算固定，优先复用开源服务（Supabase / Vercel）
- 数据透明：无论流量还是收入，定期公开

## 交付节奏

1. 第 1 周：需求冷启动 + UI 草图  
2. 第 2 周：核心功能上线 Preview 环境  
3. 第 3 周：写文档与自动化测试  
4. 第 4 周：正式版 + 博客 + 社区发帖

<AccentCallout title="测量什么" tone="green">
  留存、转化、反馈数量比 PV 更重要。常用 Supabase SQL + Vercel Analytics。
</AccentCallout>
